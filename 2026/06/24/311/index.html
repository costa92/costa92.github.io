claude-code-setup：让 Claude 扫描你的项目，自动给出最优配置方案

用了 Claude Code 一段时间之后，你可能遇到过这个问题：明明知道可以配置很多东西，但不知道从哪开始，也不确定什么配置对当前这个项目最有价值。

钩子、MCP 服务器、子代理、技能包——这些概念散落在文档的不同角落，每个都要单独去研究。加上不同项目的技术栈不一样，”什么配置对我这个项目有用”这个问题没有一个通用的回答。对于刚接触 Claude Code 的人来说，这段摸索的成本比预期高很多。

claude-code-setup 做的事情很具体：扫描你的代码库，根据项目实际用到的语言、框架和依赖，给出针对性的配置建议。不是给你一份通用清单，是真的看过你的项目再说话。

这是 Anthropic 官方出品的插件（claude-plugins-official 市场中的 claude-code-setup），作者是 Anthropic 内部团队，当前版本 1.0.0。

安装只需一条命令

Claude Code 启动时会自动加载官方市场，不需要额外添加源。在 Claude Code 中运行：

/plugin install claude-code-setup@claude-plugins-official

安装后激活插件：

/reload-plugins

如果提示找不到插件，多半是市场没加载或过期了。先刷新一下市场再重试：

/plugin marketplace update claude-plugins-official

刷新后还找不到，可能是这台机器从没加过这个市场，手动加一次：

/plugin marketplace add anthropics/claude-plugins-official

要是插件装上了、技能却死活不出现，通常是缓存坏了——清掉缓存目录，重启 Claude Code 再装一遍就好：

rm -rf ~/.claude/plugins/cache

我在 MacBook 上测试全程不超过 30 秒。安装支持三种范围：用户级（跨所有项目生效）、项目级（写入 .claude/settings.json，可以共享给团队协作者）、本地级（只在当前仓库生效，不共享）。我一般选用户级，让它在所有项目里都能随时调用。

如果你习惯用图形界面，也可以在 Claude Code 里运行 /plugin 打开插件管理器，切到 Discover 标签页，搜索 claude-code-setup 然后点安装。两种方式结果一样，图形界面里还会提前显示这个插件会加载哪些内容和每轮对话大概增加多少上下文成本（Claude Code 2.1.143 以后版本开始显示这个数字），可以提前评估。

安装完之后不需要做任何额外配置，切到你的项目目录就能直接用。值得一提的是，这个插件是只读模式——它只分析，不修改任何文件，不写 .claude/settings.json，不动 CLAUDE.md，所有建议都以文本形式输出给你看，装不装完全由你决定。这对于还不太熟悉 Claude Code 配置机制的人来说，安全边界很清晰。

反过来，装了之后想撤也简单：运行 /plugin 切到 Installed 标签页就能启用、停用或卸载，命令行直接 /plugin uninstall claude-code-setup@claude-plugins-official 也行。试错成本很低，不合用随时移除。

它扫了什么、推了什么

我装好后直接用自然语言触发：

recommend automations for this project

或者更简短的：

help me set up Claude Code

插件会先读取项目结构，分析用到了什么语言、有没有前端框架、依赖里有没有数据库 SDK 或外部 API 客户端，然后给出 5 类推荐，每类最多 2 条。

MCP 服务器：外部工具集成，把 Claude 连接到数据库、API、文档系统等。比如项目里有大量调用外部文档的需求，会推 context7（实时查官方文档，避免 Claude 凭记忆讲过时 API）；有前端 UI 测试，会推 Playwright MCP（让 Claude 直接操控浏览器做验证）。

技能包：预打包的专项能力，一条命令装好就能用。Plan agent 可以让 Claude 在动手之前先输出完整的修改计划，适合改动涉及多个文件的任务；frontend-design 技能提供了一套针对 UI 层的代码约定和审查标准。

钩子：绑定到 Claude 工具事件上的自动动作，不需要每次手动触发。比如每次 Claude 编辑完文件后自动跑格式化，或者在 Claude 试图写入某些敏感路径时主动阻断并提示风险——这类保护在多人协作的项目里特别有用。

子代理：专项检查的后台代理，可以和主任务并行跑。安全子代理会在每次改动之后扫描常见漏洞类型；性能子代理关注资源占用；无障碍子代理专门检查前端输出是否符合 WCAG 规范，省掉手动审查的时间。

斜杠命令：预定义的快捷工作流，比如 /pr-review 自动审查当前改动的质量和风险，/test 快速生成测试用例，/explain 对当前文件生成解释性注释。这些工作流在不同项目里重复用，打包成命令比每次重新描述需求效率高很多。

每类推荐最多 2 条是有意为之，不想在你第一次用的时候就堆太多选项。如果想深入某一类，可以针对具体类型再追问，插件会在那个类别里给出更多选项。

类型解决的问题典型例子
MCP 服务器Claude 需要访问外部资源context7、Playwright
技能包特定任务需要专项能力Plan agent、frontend-design
钩子重复动作自动执行自动格式化、拦截敏感路径
子代理需要专项检查安全、性能、无障碍审查
斜杠命令常用工作流一键触发/pr-review、/test

实际效果和一个重要前提

我在几个不同的项目上试了一遍，推荐结果和项目实际情况的对应关系都挺准的。

我那个 Next.js 项目里有 Playwright 测试用例，推荐里就真冒出了 Playwright MCP；package.json 里声明了 TypeScript，它就给了 typescript-lsp（代码跳转和类型检查）。我又换到一个 Go 微服务仓库，推荐立马变成了 gopls-lsp 和 git commit 工作流技能。并没有给出什么通用的”适合所有项目”的结果，确实是在看你的代码再说的。

拿到建议之后，按需安装对应的插件：

# 例：安装 context7（官方文档 MCP 服务器）
/plugin install context7@claude-plugins-official
# 例：安装 gopls-lsp（Go 语言服务器）
/plugin install gopls-lsp@claude-plugins-official

装 LSP 类插件有个前提：像 gopls-lsp、typescript-lsp 这类代码智能插件，本身只配置 Claude Code 的 LSP 连接，真正干活的语言服务器二进制得你自己装好——gopls-lsp 依赖 gopls，typescript-lsp 依赖 typescript-language-server。装完插件如果在 /plugin 的 Errors 标签页看到 Executable not found in $PATH，就是少了对应二进制，补上即可。装好之后 Claude 每次改完文件会自动拿到类型错误和缺失 import，不用手动跑编译器（按 Ctrl+O 可内联查看诊断）。

有一点值得说清楚：claude-code-setup 只给建议，不帮你执行。扫完出一份清单，装不装、装哪个，全由你自己决定。这个设计让你可以逐条评估，也可以直接追问 Claude “为什么推荐这个”来了解背后的判断依据，比直接帮你全装上要透明很多。我试过追问几次，解释通常都很清楚——比如”你的 package.json 里有 @playwright/test，所以推了 Playwright MCP”，而不是模糊地说”因为你有测试需求”。

不太适合的情况我也得说清楚：如果你已经系统用过 Claude Code 的配置体系，这个插件提供的信息密度有限——推荐范围主要基于已知的常见模式，不会去搜索市场上所有可用插件，也不会针对特别小众的框架给出定制建议。对于想要”分析完直接帮我全装上”的需求，还是得自己再操作一步。

适合的场景：刚开始用 Claude Code 不知道从哪里配置；接手了别人的项目或换了新的技术栈，想快速了解有没有现成的插件可以用；团队里来了新人，需要给出配置起点，省掉每个人重新摸索的时间。

配置之后的下一步

切到你在用的项目目录，对 Claude 说：

what hooks should I use?

看看它对这个项目的具体建议。有用的装进来，不相关的跳过，比从零开始猜配置效率高很多。

如果你想集中了解某一类，也可以直接问得更具体，比如”给我推荐适合这个项目的 MCP 服务器”或者”有没有合适的安全检查子代理”——插件支持这类聚焦请求，会在单一类别里给出更多选项（3-5 条而不是 2 条上限）。

对某条推荐不确定要不要装，直接追问”为什么推荐这个”。它会给出具体依据，比如”你的 package.json 里有 @playwright/test，所以推了 Playwright MCP”，而不是模糊地说”因为你有测试需求”。根据这个逻辑判断值不值得装，比盲目试要靠谱很多。

插件官方文档在 Anthropic 的 claude-plugins-official 仓库，README 里有每个推荐类型的解释。更多可用的官方插件可以在 Claude Code 的 discover-plugins 文档 里按类别浏览——代码智能（各语言 LSP）、外部集成（GitHub、Figma、Slack 等）、自动安全审查（security-guidance 插件）都在里面。官方市场由 Anthropic 策展、是否收录由官方把关，来源比第三方市场可靠；不过官方也明确提醒：插件会以你的权限执行代码，Anthropic 并不保证每个插件都按预期工作，所以装前最好在详情页扫一眼它”会装哪些东西”（v2.1.145 起详情页会列出插件包含的命令、技能、MCP/LSP）再决定。

配置这种事情，实际跑一遍才知道哪些真正有用。claude-code-setup 省掉的是”我都不知道有这些可以配”的那段时间，先让 Claude 给出一份可以执行的候选清单，再按需决策。

收藏起来，下次换项目配置 Claude Code 时按图索骥。
