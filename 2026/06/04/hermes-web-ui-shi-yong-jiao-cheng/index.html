hermes-web-ui 使用教程

hermes-web-ui 是 Hermes Agent 的全功能 Web 管理面板，可以通过简洁、响应式的 Web 界面管理 AI 对话会话、监控使用情况与成本、配置平台渠道、安排定时任务、浏览技能——所有操作都在一个干净的界面中完成。

什么是 Hermes Agent

Hermes Agent 是一个多平台 AI 代理，支持 Telegram、Discord、Slack、WhatsApp 等多个平台。hermes-web-ui 为其提供了可视化的管理界面，无需命令行即可完成大多数配置操作。

项目信息：

  • GitHub: EKKOLearnAI/hermes-studio
  • 许可证: BSL-1.1

快速开始

方法一：npm 安装（推荐）

# 安装
npm install -g hermes-web-ui

# 启动
hermes-web-ui start

启动后打开 http://localhost:8648

方法二：一键安装脚本（自动检测系统）

支持 Debian/Ubuntu/macOS，自动安装 Node.js（如缺少）和 hermes-web-ui：

bash <(curl -fsSL https://cdn.jsdelivr.net/gh/EKKOLearnAI/hermes-web-ui@main/scripts/setup.sh)

方法三：Docker Compose

# 使用预构建镜像（推荐）
WEBUI_IMAGE=ekkoye8888/hermes-web-ui:latest docker compose up -d hermes-agent hermes-webui

# 或从源码构建
docker compose up -d --build hermes-agent hermes-webui

# 查看日志
docker compose logs -f hermes-webui

打开 http://localhost:8648

Docker 数据持久化：

  • Hermes 数据存储在 ./hermes_data
  • Web UI 认证 token 存储在 ./hermes_data/hermes-web-ui/.token
  • 首次运行启用认证时，token 会打印在容器日志中

WSL 环境

# 安装
bash <(curl -fsSL https://cdn.jsdelivr.net/gh/EKKOLearnAI/hermes-web-ui@main/scripts/setup.sh)

# 启动
hermes-web-ui start

# WSL 会自动检测并使用 hermes gateway run 后台启动
hermes gateway run

主要功能

AI 对话

  • 实时流式响应：通过 SSE 实现，支持异步运行
  • 多会话管理：创建、重命名、删除、切换会话
  • 自建会话数据库：本地 SQLite 存储，首次启动时自动从 Hermes state.db 同步会话
  • 会话分组：按来源（Telegram、Discord、Slack 等）分组，可折叠
  • 实时指示器：活跃会话置顶显示，带旋转图标
  • Markdown 渲染：语法高亮 + 代码复制
  • 工具调用详情：展开查看参数和结果
  • 文件上传/下载：支持本地、Docker、SSH、Singularity 后端
  • 全局搜索：Ctrl+K 跨所有对话搜索
  • 全局模型选择器：从 ~/.hermes/auth.json 凭据池发现模型
  • 每会话模型标签：显示当前使用的模型和上下文 Token 用量
# 凭据池位置
~/.hermes/auth.json

平台渠道配置

统一管理 8 个平台，在一个页面完成所有配置：

  • 凭据管理：写入 ~/.hermes/.env
  • 渠道行为设置：写入 ~/.hermes/config.yaml
  • 配置变更自动重启网关
  • 各平台配置状态检测
# 配置文件位置
~/.hermes/.env        # 凭据
~/.hermes/config.yaml # 渠道行为配置

使用分析

  • Token 使用量分解：输入 / 输出分别统计
  • 会话数量与日均
  • 预估成本追踪 & 缓存命中率
  • 模型使用分布图
  • 30 天每日趋势：柱状图 + 数据表

定时任务

  • 完整的 Cron 管理：创建、编辑、暂停、恢复、删除
  • 立即触发执行
  • Cron 表达式快捷预设

模型管理

  • 自动发现模型：从凭据池 ~/.hermes/auth.json 读取
  • 获取提供商模型列表：调用 /v1/models 接口
  • 添加、更新、删除提供商：预设 & 自定义 OpenAI 兼容接口
  • OpenAI Codex & Nous Portal OAuth 登录
  • 提供商 URL 自动检测：支持非 v1 版本（如 /v4）
  • 提供商级别模型分组：可切换默认模型
# API 版本示例
/v1/models  # 标准
/v4/models  # 非标准版本自动检测

多配置文件 & 网关

  • 配置文件管理：创建、重命名、删除、切换
  • 克隆或导入：从归档文件（.tar.gz）导入
  • 导出备份：分享或备份配置
  • 多网关管理：每个配置文件独立启动、停止、监控网关
  • 端口冲突自动解决
  • 配置文件作用域隔离

文件浏览器

  • 远程浏览：支持本地、Docker、SSH、Singularity 后端
  • 完整文件操作：上传、下载、重命名、复制、移动、删除
  • 创建目录
  • 文件预览：语法高亮

群聊

  • 多代理聊天室：通过 Socket.IO 实时消息
  • @提及路由：提及代理触发上下文回复
  • 上下文压缩：历史超过 Token 阈值自动摘要
  • typing 状态和回复进度指示
  • 房间管理：创建、删除、邀请码
  • 代理管理：添加/移除代理，带独立配置文件
  • SQLite 消息持久化
  • 移动端响应式：侧边栏可折叠

技能与记忆

  • 浏览和搜索已安装技能
  • 查看技能详情和附加文件
  • 用户笔记和配置文件管理

日志

  • 查看代理 / 网关 / 错误日志
  • 多维度过滤：日志级别、文件、关键词
  • 结构化日志解析：HTTP 访问日志高亮

认证

  • Token 认证：首次运行自动生成，或通过 AUTH_TOKEN 环境变量设置
  • 可选用户名/密码登录：首次 Token 认证后通过设置页面设置
  • 禁用认证：AUTH_DISABLED=1
# 环境变量
AUTH_TOKEN=your-token    # 设置认证 token
AUTH_DISABLED=1         # 禁用认证（不推荐）

设置

  • 显示：流式输出、紧凑模式、推理、成本显示
  • 代理：最大轮次、超时、工具强制
  • 记忆：启用/禁用、字符限制
  • 会话重置：空闲超时、计划重置
  • 隐私：PII 去标识化
  • 模型设置：默认模型 & 提供商
  • API 服务器配置

Web Terminal

  • 集成终端：基于 node-pty 和 @xterm/xterm
  • 多会话支持：创建、切换、关闭终端会话
  • 实时键盘输入：通过 WebSocket 流式传输 PTY 输出
  • 窗口大小调整支持

CLI 命令

# 启动
hermes-web-ui start

# 指定端口
hermes-web-ui start --port 9000

# 停止
hermes-web-ui stop

# 重启
hermes-web-ui restart

# 查看状态
hermes-web-ui status

# 更新
hermes-web-ui update

# 版本
hermes-web-ui -v

# 帮助
hermes-web-ui -h

自动配置

启动时 BFF 服务器自动执行：

  1. 验证 ~/.hermes/config.yaml 并填充缺失的 api_server 字段
  2. 如配置被修改，备份原文件为 config.yaml.bak
  3. 检测并启动网关（如需要）
  4. 解决端口冲突（终止占用进程）
  5. 启动成功后自动打开浏览器
# 配置文件位置
~/.hermes/config.yaml

# 备份文件（如有修改）
config.yaml.bak

开发指南

# 克隆源码
git clone https://github.com/EKKOLearnAI/hermes-studio.git
cd hermes-studio

# 安装依赖
npm install

# 开发模式
npm run dev
  • 前端开发服务器：http://localhost:5173
  • BFF 服务器：http://localhost:8648（代理到 Hermes 的 8642 端口）
# 生产构建
npm run build   # 输出到 dist/

技术架构

Browser → BFF (Koa, :8648) → Hermes Gateway (:8642)
                ↓
           Hermes CLI (sessions, logs, version)
                ↓
           ~/.hermes/config.yaml  (渠道行为)
           ~/.hermes/auth.json    (凭据池)
           Tencent iLink API      (微信扫码登录)

架构说明：

  • 前端：Vue 3 + TypeScript + Vite + Naive UI + Pinia + Vue Router + vue-i18n + SCSS + markdown-it + highlight.js
  • BFF 层：Koa 2，处理 API 代理（路径重写）、SSE 流式响应、文件上传下载（多后端：本地/Docker/SSH/Singularity）、会话 CRUD、配置/凭据管理、微信扫码登录、模型发现、技能/记忆管理、日志读取、静态文件服务
  • Web Terminal：node-pty + @xterm/xterm

前端采用多代理可扩展设计，所有 Hermes 相关代码都命名空间在 hermes/ 目录下（API、组件、视图、存储），便于添加新的代理集成。

常见问题

首次启动需要认证？

首次运行时会自动生成认证 token，并在终端打印。按提示访问 http://localhost:8648 并输入 token 登录。

Docker 环境下无法使用 hermes cli？

确保 docker-compose.yml 正确配置了 hermes-agent 服务，且网络连接正常。

端口被占用？

hermes-web-ui 会自动检测并解决端口冲突，终止占用端口的旧进程。

如何查看详细日志？

在 Web UI 的「日志」页面可查看代理、网关、错误日志，支持按级别和关键词过滤。

总结

hermes-web-ui 为 Hermes Agent 提供了一个功能完整的 Web 管理界面，覆盖了从日常对话管理到高级配置的所有场景。无论是个人使用还是团队协作，都能显著提升体验。

推荐安装方式：

  • 本地开发：npm 安装
  • 生产环境：Docker Compose

立即体验：

npm install -g hermes-web-ui && hermes-web-ui start
