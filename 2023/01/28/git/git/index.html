git 安装

git
Tool
发布日期:   2023-01-28
文章字数:   640
阅读次数:  

最早Git是在Linux上开发的，很长一段时间内，Git也只能在Linux和Unix系统上跑。不过，慢慢地有人把它移植到了Windows上。现在，Git可以在Linux、Unix、Mac和Windows这几大平台上正常运行了。

Mac install

brew install git

Ubuntu install

sudo apt install git

git 提交标准

git 常用命令

查看代码版本：

git describe --always --tags --abbrev=0 --match 'v*' --exclude '*/*' | tr -d "[\r\n]"

删除tag

git tag --delete  tag name  // v本地
git push origin :refs/tags/v1.0.0  // 删除远程 v1.0.0

修改tag 名称

git tag new_tag old_tag
git tag -d old_tag
git push origin :refs/tags/old_tag
git push --tags

删除远程分支

git push origin --delete

强制切换到 commit_id 这个记录

git reset --hard commit_id

强制提交到远程

git push -f

删除本地的远程不存在的分支

git remote prune origin

git commit完之后，还没有执行git push，想修改/撤销这个commit，怎么办？
如果只是想修改注释，可以这样操作

git commit --amend
这个时候进入`vim`编辑，直接修改即可，修改完注释，退出`vim`编辑
`:wq`保存已编辑的注释，重新`git push`即可

要撤回commit

git reset --soft HEAD^
这样就能成功的撤回你刚刚的`commit`操作。
HEAD^的意思是上一个版本，也可以写成HEAD~1
如果你进行了2次commit，想都撤回，可以使用HEAD~2

注意，这个命令仅仅是撤回commit操作，写的代码仍然保留

恢复本地未提交内容

git checkout .

git reset HEAD  *
# 回退到上一个版本
git reset --hard HEAD^
#回退到上上次版本
git reset --hard HEAD^^
git reset --hard HEAD^^^

#回退到指定commitid的版本
git reset --hard  commit_id

恢复本地已提交内容

git reset --hard

恢复远程未提交内容

git fetch origin
git reset --hard origin/master

git 常见的错误处理

1、fatal: Not possible to fast-forward, aborting.

出现的原因：

两个分之同时改了同样的地方，造成冲突。一般的情况使用 merge 合并分支

git checkout master
git merge dev

如果出现代码冲突的时候需要手动解决，但是有时候是无法合并成功的。

解决:

git checkout master
git rebase dev

如果是一个分支，但是代码出现修改，也会出现改错误。

解决:

git pull origin master --rebase

git pull origin development --rebase

git pull origin test --rebase

注意 修改冲突代码

在提交代码

git push origin HEAD:develop     // develop 是分支

在冲突解决完毕并且提交代码后，执行下面的命令：

// 在终端也会有需要执行这个命令的提示
git rebase --continue
